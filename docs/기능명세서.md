# 먹픽 기능명세서 초안

## 1. 프로젝트 개요

### 1.1 프로젝트명

먹픽 (MUK PICK)

### 1.2 프로젝트 주제

개인 및 집단의 음식 선호도를 분석하여 메뉴 결정 갈등을 줄여주는 앱

### 1.3 프로젝트 목적

먹픽은 개인과 단체가 식사 메뉴를 더 빠르고 만족스럽게 결정할 수 있도록 돕는 것을 목적으로 한다. 개인 사용자는 일상적인 식사 상황에서 자신의 음식 취향과 선호도에 맞는 메뉴를 추천받을 수 있다. 또한 단체 사용자는 약속이나 모임에서 서로 다른 취향으로 인해 발생하는 메뉴 선택 갈등을 줄일 수 있다.

먹픽은 단순한 랜덤 메뉴 추천 방식이 아니라, 입력된 사용자의 선호 음식, 기피 음식, 선호 카테고리 등의 정보를 바탕으로 상황에 맞는 메뉴를 추천한다. 특히 집단 약속에서는 참여자들의 개인 선호도를 종합적으로 반영하여, 모두가 수용하기 쉬운 메뉴 후보를 제안한다.

### 1.4 기대 효과

- 메뉴 선택 과정에서 불필요한 고민과 대기 시간을 줄일 수 있다.
- 개인의 음식 취향이 추천 결과에 반영되어 사용자 만족도를 높일 수 있다.
- 단체 모임에서 특정 사람의 의견에 치우치지 않는 균형 잡힌 메뉴 선택이 가능하다.
- 구성원 간 선호도 차이로 인한 갈등을 줄이고 의사결정 과정을 원활하게 만들 수 있다.
- 익숙한 메뉴뿐만 아니라 새로운 메뉴 후보를 제안하여 식사 선택의 다양성을 높일 수 있다.

## 2. 예상 사용자

### 2.1 개인 사용자

혼자 식사할 때 어떤 메뉴를 먹을지 결정하기 어려운 사용자이다. 자신의 선호도와 선택 조건을 기반으로 메뉴를 추천받을 수 있다.

### 2.2 모임 생성자

친구, 팀원, 동료 등과의 식사 약속을 만들고 참여자를 초대하는 사용자이다. 모임의 날짜, 인원, 식사 목적 등을 설정하고 모임 메뉴 추천을 요청할 수 있다.

### 2.3 모임 참여자

생성된 모임에 참여하여 자신의 음식 선호도, 기피 음식, 알레르기 등을 반영하는 사용자이다. 추천된 메뉴 후보를 확인할 수 있다.

## 3. 전체 서비스 흐름

### 3.1 개인 메뉴 추천 흐름

1. 사용자가 앱에 로그인한다.
2. 사용자가 자신의 음식 선호도와 제한 조건을 설정한다.
3. 사용자가 개인 메뉴 추천 기능을 선택한다.
4. 시스템은 사용자의 선호도, 기피 음식, 알레르기 정보 등을 확인한다.
5. 시스템은 적합한 메뉴 후보를 추천한다.
6. 사용자는 추천 메뉴 중 하나를 선택한다.
7. 선택된 메뉴는 식사 기록에 저장될 수 있다.

### 3.2 모임 메뉴 추천 흐름

1. 모임 생성자가 식사 약속을 생성한다.
2. 모임 생성자가 참여자를 초대한다.
3. 참여자들은 자신의 선호도와 제한 조건을 선택한다.
4. 시스템은 참여자들의 선호도를 종합하여 모임 선호도를 분석한다.
5. 시스템은 모임 전체가 수용하기 쉬운 메뉴 후보를 추천한다.
6. 참여자들은 추천 메뉴 후보를 확인한다.
7. 선택된 메뉴는 모임 기록에 저장될 수 있다.

## 4. 핵심 기능 목록

먹픽의 핵심 기능은 사용자 관리, 개인 선호도 관리, 개인 메뉴 추천, 모임 관리, 모임 메뉴 추천, 식사 및 추천 기록 관리, 메뉴 데이터 관리 기능으로 구분한다.

### 4.1 사용자 관리 기능

| 기능 ID | 기능명 | 설명 |
|---|---|---|
| U-01 | 회원가입 | 사용자가 이메일, 비밀번호, 닉네임을 입력하여 계정을 생성한다. |
| U-02 | 로그인 및 로그아웃 | 사용자가 계정 정보를 입력하여 서비스에 접속하고, 사용을 마친 뒤 계정에서 로그아웃한다. |
| U-03 | 사용자 정보 관리 | 사용자가 닉네임, 기본 정보, 계정 정보를 확인하거나 수정한다. |

### 4.2 개인 선호도 관리 기능

| 기능 ID | 기능명 | 설명 |
|---|---|---|
| P-01 | 선호도 선택 | 사용자가 제공된 음식 카테고리, 음식 종류, 맛의 강도 등의 항목 중 자신의 취향에 맞는 조건을 선택한다. |
| P-02 | 제한 조건 선택 | 사용자가 기피 음식, 알레르기 재료, 종교적 제한 등 추천에서 제외해야 할 조건을 제공된 항목 중에서 선택한다. |
| P-03 | 선호도 관리 | 사용자가 선택한 선호도와 제한 조건을 조회하거나 수정한다. |

### 4.3 개인 메뉴 추천 기능

| 기능 ID | 기능명 | 설명 |
|---|---|---|
| R-01 | 개인 맞춤 메뉴 추천 | 사용자가 개인 식사 상황에서 메뉴 추천을 요청하면, 설정된 음식 선호도와 제한 조건을 바탕으로 적합한 메뉴 후보를 추천한다. |
| R-02 | 새로운 메뉴 제안 | 사용자의 선호도와 크게 벗어나지 않는 범위에서 평소 선택하지 않았던 새로운 메뉴 후보를 함께 제안한다. |
| R-03 | 추천 결과 제공 | 추천된 메뉴 후보와 간단한 추천 이유를 사용자에게 제공한다. |

### 4.4 모임 관리 기능

| 기능 ID | 기능명 | 설명 |
|---|---|---|
| G-01 | 모임 생성 | 사용자가 식사 약속을 만들고 모임 이름, 날짜, 시간 등 기본 정보를 설정한다. |
| G-02 | 참여자 초대 | 모임 생성자가 다른 사용자를 모임에 초대할 수 있다. |
| G-03 | 모임 참여 | 초대받은 사용자가 모임에 참여하고, 해당 모임에서 반영할 선호 조건을 선택할 수 있다. |
| G-04 | 모임 정보 조회 | 사용자가 모임 기본 정보, 참여자 목록, 추천 진행 상태를 확인할 수 있다. |
| G-05 | 모임 정보 수정 | 모임 생성자가 모임의 날짜, 시간, 참여자 등 일부 정보를 수정할 수 있다. |

### 4.5 모임 메뉴 추천 기능

| 기능 ID | 기능명 | 설명 |
|---|---|---|
| GM-01 | 모임 조건 반영 | 모임 참여자들이 설정한 음식 선호도, 기피 음식, 알레르기 등 메뉴 추천에 필요한 조건을 반영한다. |
| GM-02 | 모임 선호도 분석 | 참여자들의 선호 조건을 종합하여 해당 모임에 적합한 메뉴 기준을 분석한다. |
| GM-03 | 모임 메뉴 후보 추천 | 분석 결과를 바탕으로 특정 참여자에게 지나치게 불리하지 않고, 모임 전체가 수용하기 쉬운 메뉴 후보를 추천한다. |
| GM-04 | 추천 결과 제공 | 추천된 메뉴 후보와 해당 메뉴가 모임에 적합한 이유를 함께 제공한다. |

### 4.6 식사 및 추천 기록 관리 기능

| 기능 ID | 기능명 | 설명 |
|---|---|---|
| H-01 | 선택 메뉴 저장 | 개인 추천 또는 모임 추천을 통해 선택된 메뉴를 기록으로 저장한다. |
| H-02 | 추천 결과 저장 | 시스템이 제공한 추천 메뉴 후보와 사용자의 최종 선택 결과를 저장한다. |
| H-03 | 식사 기록 조회 | 사용자가 이전에 선택한 메뉴와 모임에서 결정된 메뉴 기록을 확인할 수 있다. |
| H-04 | 만족도 기록 | 사용자가 선택한 메뉴에 대한 만족도를 남기고, 해당 정보를 이후 추천에 참고할 수 있도록 한다. |

### 4.7 메뉴 데이터 관리 기능

| 기능 ID | 기능명 | 설명 |
|---|---|---|
| M-01 | 메뉴 데이터 관리 | 추천에 사용할 음식 메뉴 목록을 등록하고 관리한다. |
| M-02 | 메뉴 카테고리 관리 | 메뉴를 한식, 중식, 일식, 양식, 분식 등 음식 카테고리별로 분류한다. |
| M-03 | 메뉴 속성 관리 | 메뉴의 매운 정도, 주요 재료, 고기·해산물 포함 여부 등 추천에 필요한 속성 정보를 관리한다. |
| M-04 | 추천 제외 조건 관리 | 알레르기, 기피 음식, 제한 조건과 관련된 메뉴 정보를 관리하여 부적절한 메뉴가 추천되지 않도록 한다. |

## 5. 기능별 상세 명세

### 5.1 사용자 관리 기능

- **입력**
  - 이메일
  - 비밀번호
  - 닉네임
  - 수정할 사용자 기본 정보

- **처리**
  - 사용자는 회원가입을 통해 계정을 생성한다.
  - 사용자는 이메일과 비밀번호를 사용해 로그인하거나 로그아웃할 수 있다.
  - 시스템은 회원가입 시 입력값의 형식과 중복 여부를 확인한다.
  - 시스템은 로그인 시 계정 정보의 일치 여부를 검사한다.
  - 사용자가 닉네임 등 기본 정보를 수정할 경우, 변경된 정보의 유효성을 확인한 뒤 저장한다.

- **출력**
  - 회원가입, 로그인, 로그아웃, 사용자 정보 수정 결과를 메시지로 제공한다.
  - 처리 결과에 따라 메인 화면, 로그인 화면, 사용자 정보 화면 등으로 이동한다.

- **예외 상황**
  - 이메일 형식이 올바르지 않은 경우 오류 메시지를 표시한다.
  - 이미 가입된 이메일인 경우 오류 메시지를 표시한다.
  - 비밀번호가 조건에 맞지 않는 경우 다시 입력하도록 안내한다.
  - 로그인 정보가 일치하지 않는 경우 다시 입력하도록 안내한다.
  - 사용자 정보 수정에 실패한 경우 저장되지 않았음을 안내한다.

### 5.2 개인 선호도 관리 기능

- **입력**
  - 음식 카테고리
  - 음식 종류
  - 맛의 강도
  - 기피 음식
  - 알레르기 재료
  - 종교적 제한 재료
  - 기타 제한 조건

- **처리**
  - 사용자는 앱에서 제공하는 항목 중 자신의 음식 취향에 맞는 조건을 선택한다.
  - 사용자는 기피 음식, 알레르기 재료, 종교적 제한 등 추천에서 제외해야 할 제한 조건을 선택한다.
  - 시스템은 선택된 선호도와 제한 조건을 사용자 프로필에 저장한다.
  - 저장된 선호도 정보는 개인 메뉴 추천과 모임 메뉴 추천에 활용된다.
  - 사용자는 저장된 선호도 정보를 조회하거나 수정할 수 있다.

- **출력**
  - 선택된 음식 선호도 정보를 표시한다.
  - 선택된 기피 음식, 알레르기 정보, 제한 조건 목록을 표시한다.
  - 선호도 설정 또는 수정이 완료되면 저장 완료 메시지를 제공한다.

- **예외 상황**
  - 사용자가 선호도 정보를 선택하지 않은 경우 기본 추천 조건을 사용하거나 선호도 설정을 안내한다.
  - 알레르기 등 필수 제한 조건이 있는 경우 해당 조건과 충돌하는 메뉴는 추천에서 제외한다.
  - 선호도 저장 과정에서 오류가 발생하면 다시 시도하도록 안내한다.

### 5.3 개인 메뉴 추천 기능

- **입력**
  - 사용자의 음식 선호도
  - 기피 음식
  - 알레르기 정보
  - 선택한 음식 카테고리
  - 맛의 강도 등 추천 조건

- **처리**
  - 사용자가 개인 메뉴 추천을 요청하면 시스템은 사용자의 선호도와 제한 조건을 확인한다.
  - 시스템은 추천 가능한 메뉴 후보를 탐색한다.
  - 기피 음식이나 알레르기 조건과 충돌하는 메뉴는 제외한다.
  - 사용자의 선호도와 잘 맞는 메뉴를 우선 추천한다.
  - 사용자의 기존 선호도와 크게 벗어나지 않는 범위에서 새로운 메뉴 후보도 함께 제안할 수 있다.

- **출력**
  - 개인에게 적합한 추천 메뉴 후보를 제공한다.
  - 각 메뉴에 대한 간단한 추천 이유를 제공한다.
  - 필요한 경우 새로운 메뉴 후보도 함께 표시한다.

- **예외 상황**
  - 추천 가능한 메뉴가 부족한 경우 일부 조건을 완화하거나 다른 카테고리 선택을 안내한다.
  - 사용자의 선호도 정보가 부족한 경우 기본 메뉴 후보를 제공하거나 선호도 설정을 먼저 진행하도록 안내한다.
  - 알레르기 조건과 충돌하는 메뉴는 추천 결과에 표시하지 않는다.

### 5.4 모임 관리 기능

- **입력**
  - 모임 이름
  - 날짜
  - 시간
  - 참여자 정보
  - 모임에서 반영할 선호 조건

- **처리**
  - 모임 생성자는 식사 약속을 생성하고 모임의 기본 정보를 설정한다.
  - 시스템은 생성된 모임 정보를 저장한다.
  - 시스템은 참여자를 초대할 수 있는 기능을 제공한다.
  - 초대받은 사용자는 모임에 참여할 수 있다.
  - 참여자는 해당 모임에서 반영할 음식 선호도나 제한 조건을 선택할 수 있다.
  - 사용자는 모임 정보, 참여자 목록, 추천 진행 상태를 확인할 수 있다.

- **출력**
  - 모임 생성 완료 메시지를 표시한다.
  - 초대 정보를 제공한다.
  - 모임 상세 정보, 참여자 목록, 추천 진행 상태를 표시한다.

- **예외 상황**
  - 모임 이름, 날짜, 시간 등 필수 정보가 누락된 경우 모임 생성을 제한한다.
  - 초대받은 사용자가 참여하지 않은 경우 해당 사용자를 제외하고 추천을 진행하거나 대기 상태로 표시한다.
  - 모임 정보 수정 권한이 없는 사용자가 수정을 시도할 경우 수정할 수 없도록 안내한다.

### 5.5 모임 메뉴 추천 기능

- **입력**
  - 모임 참여자들의 음식 선호도
  - 기피 음식
  - 알레르기 정보
  - 음식 카테고리
  - 메뉴 데이터

- **처리**
  - 시스템은 모임 참여자들이 설정한 선호도와 제한 조건을 종합한다.
  - 시스템은 해당 모임에 적합한 메뉴 기준을 분석한다.
  - 알레르기나 강한 기피 조건과 충돌하는 메뉴는 제외한다.
  - 특정 참여자에게 지나치게 불리한 메뉴가 추천되지 않도록 조정한다.
  - 모임 전체가 수용하기 쉬운 메뉴 후보를 우선적으로 추천한다.

- **출력**
  - 모임에 적합한 메뉴 후보 목록을 제공한다.
  - 각 메뉴가 추천된 간단한 이유를 함께 제공한다.
  - 추천 결과는 모임 참여자들이 확인할 수 있도록 모임 화면에 표시한다.

- **예외 상황**
  - 참여자들의 선호 조건이 크게 충돌하는 경우 공통적으로 제외해야 하는 조건을 우선 반영한다.
  - 선호도 충돌이 큰 경우 비교적 중립적인 메뉴 후보를 제안한다.
  - 참여자 정보가 부족한 경우 현재까지 입력된 정보를 기준으로 임시 추천을 제공한다.
  - 모든 조건을 만족하는 메뉴가 없는 경우 조건 완화가 필요함을 안내한다.

### 5.6 식사 및 추천 기록 관리 기능

- **입력**
  - 개인 추천 또는 모임 추천을 통해 선택된 메뉴
  - 추천 메뉴 후보
  - 최종 선택 결과
  - 만족도 정보

- **처리**
  - 시스템은 사용자가 선택한 메뉴와 추천 결과를 기록으로 저장한다.
  - 개인 추천과 모임 추천에서 선택된 메뉴를 구분하여 저장한다.
  - 사용자가 이전 기록을 확인할 수 있도록 관리한다.
  - 선택적으로 사용자가 만족도를 남기면 해당 정보를 이후 추천 개선에 참고할 수 있다.

- **출력**
  - 사용자의 이전 선택 메뉴를 제공한다.
  - 모임에서 결정된 메뉴 기록을 제공한다.
  - 추천 기록과 만족도 기록을 조회할 수 있도록 제공한다.
  - 메뉴 선택이나 만족도 저장이 완료되면 저장 완료 메시지를 표시한다.

- **예외 상황**
  - 기록 저장에 실패한 경우 저장되지 않았음을 안내하고 다시 시도할 수 있도록 한다.
  - 기록이 없는 신규 사용자의 경우 조회할 기록이 없음을 표시한다.
  - 만족도를 입력하지 않은 경우 선택 메뉴만 저장한다.

### 5.7 메뉴 데이터 관리 기능

- **입력**
  - 메뉴명
  - 음식 카테고리
  - 매운 정도
  - 주요 재료
  - 고기·해산물 포함 여부
  - 알레르기 관련 정보
  - 추천 제외 조건

- **처리**
  - 시스템은 추천에 사용할 메뉴 데이터를 등록하고 관리한다.
  - 각 메뉴를 한식, 중식, 일식, 양식, 분식 등 음식 카테고리별로 분류한다.
  - 메뉴별 매운 정도, 주요 재료, 음식 성격 등의 속성 정보를 관리한다.
  - 알레르기나 기피 음식과 관련된 정보를 함께 관리한다.
  - 부적절한 메뉴가 추천되지 않도록 추천 제외 조건을 반영한다.

- **출력**
  - 추천에 활용 가능한 메뉴 목록을 제공한다.
  - 카테고리 정보를 제공한다.
  - 메뉴 속성 정보를 제공한다.
  - 추천 제외 조건 정보를 제공한다.

- **예외 상황**
  - 메뉴 데이터가 부족한 경우 추천 결과가 제한될 수 있음을 안내한다.
  - 메뉴 속성이나 알레르기 정보가 누락된 경우 추천 정확도가 낮아질 수 있음을 표시한다.
  - 중복 메뉴가 등록될 경우 기존 데이터와 비교하여 중복 여부를 확인한다.

## 6. 추천 로직 초안

### 6.1 개인 추천 기준

개인 메뉴 추천은 다음 요소를 반영한다.

- 사용자가 선택한 선호 음식 또는 선호 카테고리
- 사용자가 선택한 기피 음식 포함 여부
- 알레르기 또는 제한 재료 포함 여부
- 음식 카테고리 선호도
- 맛의 강도 또는 음식 속성 선호도
- 새로운 메뉴 시도 가능성

### 6.2 모임 추천 기준

모임 메뉴 추천은 다음 요소를 반영한다.

- 참여자 전체의 공통 선호도
- 참여자들이 공통적으로 선호하는 음식 카테고리
- 특정 참여자가 강하게 기피하는 음식 여부
- 알레르기나 종교적 제한 조건 여부
- 특정 참여자에게 지나치게 불리하지 않은 메뉴 여부
- 모임 전체가 수용하기 쉬운 메뉴 여부

### 6.3 추천 점수 예시

최종 추천 점수는 다음과 같은 방식으로 계산할 수 있다.

```text
최종 추천 점수
= 선호도 점수
+ 공통 선호 점수
+ 새로운 메뉴 가산점
- 기피 음식 패널티
- 최근 반복 메뉴 패널티
```

단, 알레르기나 종교적 제한 조건과 충돌하는 메뉴는 점수 감점 방식이 아니라 추천 후보에서 제외한다.

이 점수식은 초기 개발 단계에서 단순한 규칙 기반 방식으로 구현하고, 이후 사용자 데이터가 쌓이면 더 정교한 추천 방식으로 개선할 수 있다.

## 10. 비기능 요구사항

### 10.1 사용성

사용자는 복잡한 입력 없이 빠르게 메뉴를 추천받을 수 있어야 한다. 개인 추천은 가능한 한 짧은 단계로 진행되어야 하며, 모임 추천도 초대와 선호도 선택 과정이 간단해야 한다.

### 10.2 성능

추천 결과는 사용자가 오래 기다리지 않도록 빠르게 제공되어야 한다. 초기 버전에서는 규칙 기반 추천 방식을 사용하여 안정적인 응답 속도를 확보한다.

### 10.3 보안

사용자의 계정 정보와 개인 선호도 정보는 안전하게 저장되어야 한다. 비밀번호는 암호화하여 관리해야 하며, 개인 식사 기록은 다른 사용자에게 임의로 공개되지 않아야 한다.

### 10.4 개인정보 보호

음식 선호도, 알레르기 정보, 식사 기록은 개인 정보에 해당하므로 사용자의 동의 없이 외부에 노출되지 않아야 한다.

### 10.5 확장성

향후 음식점 추천, 위치 기반 추천, 배달 서비스 연동, 영양 정보 분석 등으로 확장할 수 있도록 메뉴 데이터와 추천 구조를 유연하게 설계한다.
